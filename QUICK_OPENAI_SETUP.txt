📍 OPTION 1: AUTOMATED SETUP (EASIEST)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Run this command in the project root:
  ./setup-openai.sh

Then follow the prompts!


📍 OPTION 2: MANUAL SETUP
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
1. Get API Key:
   → Go to: https://platform.openai.com/api-keys
   → Click "Create new secret key"
   → Copy the key (starts with sk-)

2. Add to Project:
   → Open: server/.env
   → Find: OPENAI_API_KEY=your_openai_api_key_here
   → Replace with: OPENAI_API_KEY=sk-your-actual-key

3. Restart Server:
   → cd server
   → npm run dev


📍 OPTION 3: USE WITHOUT OPENAI (FREE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Don't have an API key? No problem!

The AI Chat Widget works WITHOUT OpenAI using smart fallback
responses. Just leave the .env as is and it will automatically
use rule-based answers for common questions.

🆘 TROUBLESHOOTING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Problem: "OpenAI not initialized"
→ Check server/.env file exists
→ Verify API key starts with sk-
→ Restart server

Problem: "Invalid API key"
→ Generate new key from OpenAI
→ Copy entire key (very long)
→ Update .env and restart

Problem: Still not working?
→ Use fallback mode (no API key needed)
→ Check server logs for errors
→ See OPENAI_SETUP.md
